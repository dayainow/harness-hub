# 📄 제품 요구사항 정의서 (PRD: Product Requirements Document)

---

## 1. 프로젝트 개요 (Overview)
- **제품명**: Ola (올인원 AI 크리에이터 & 도구 허브)
- **비전**: 파편화된 AI 도구 정보를 한곳에 모으고, 이를 실질적으로 활용하는 '레시피(실험실)'를 통해 가장 거대한 지식 선순환 AI 생태계를 구축한다.
- **목표 (Goal)**: 
  - 신속하고 직관적인 조건별 AI 도구 다이내믹 탐색기 제공
  - 프로덕트 헌트(Product Hunt) 방식의 투표 기반 지식 경쟁 포럼 구축
  - B2C뿐 아니라 B2B 유저까지 품을 수 있는 엔터프라이즈 모니터링 환경 확보

---

## 2. 시장 분석 및 문제 정의 (Problem & Opportunity)

### 🚨 현존하는 문제 (The Problem)
1. 하루에도 약 수십 개의 AI 툴이 나오지만 일반 대중은 툴의 옥석(Freemium 여부, 카테고리)을 가려내기 힘듦.
2. 툴 디렉토리 사이트는 많이 있지만, **"그래서 이걸로 뭘 어떻게 만들수 있지?"** 라는 근본적인 Action-Plan(활용법)을 배울 장소가 없음.

### 💡 솔루션 (The Solution)
1. 카테고리, 태그, 프라이싱 등을 다중으로 엮을 수 있는 강력한 복합 필터 디렉토리.
2. 단순 게시판이 아닌 "1-depth 피드형 뷰"와 "업보트(Upvote)"가 주연인 **실험실(Market)**이라는 독립적인 생태계 설계.

---

## 3. 핵심 유저 페르소나 (Target Persona)
- **[초보자] AI 탐색러 (Novice Explorer)**: "최근에 나온 무료 영상 생성 AI 툴이 뭐가 있지?" → **다중 필터로 탐색**
- **[중급자] 프롬프트 헌터 (Prompt Hunter)**: "나도 Midjourney로 멋진 웹사이트 시안을 만들고 싶은데 방법을 알고 싶어" → **실험실 레시피 및 프롬프트 열람**
- **[전문가] 코어 크리에이터 (Core Creator)**: "내가 만든 자동화 워크플로우를 세상에 자랑해서 이목을 끌고 싶어" → **실험실 게재 및 Upvote 경쟁**

## 4. 핵심 기능 요구사항 (Functional Requirements)

| 분류 | 기능 (Feature) | 요구사항 상세 스펙 | 우선순위 |
| :--- | :--- | :--- | :---: |
| **인증** | Supabase OAuth 모듈 | 이메일 패스워드 및 글로벌 소셜 (Google 등) 1초 연동 로그인 구현 | P1 |
| **탐색** | 다이내믹 다중 배열 필터 | 프리미엄/카테고리/태그가 URL State를 기반으로 콤마(,) 기준 다중 중첩 필터링 되도록 최적화 (Prisma DB `in`, `hasSome` 기능 연계) | P1 |
| **조작** | 마이크로 이벤트 투표 (Upvote) | 실험실(Market) 컴포넌트 내부에서 리스트 이동 전파 이벤트를 막고 프론트엔드 단독 통신으로 하트/투표 카운트 구현 | P1 |
| **번역** | 글로벌 언어 팩 지원 (i18n) | `next-intl` 라우팅을 이용해 `/[locale]` 구조로 다국어 지원 | P2 |
| **백오피스** | 어드민 및 웹훅(Webhook) | 관리자 전용 대시보드 및 도구 추가 요청 시 디스코드(Discord)로 알림 라우팅 | P2 |

## 5. 비기능적 요구사항 (Non-Functional Requirements)
- **응답성 (Performance)**: 필터링 API 데이터 검색 응답 시간을 50ms 미만으로 확보할 것. 외부 에셋 로딩은 Next.js Edge Cache를 타도록 `remotePatterns` 설정 필수.
- **보안성 (Security)**: 백엔드는 `@nestjs/throttler`를 사용하여 Ddos 및 무작위 스크래핑을 차단하고, 프론트엔드는 Supabase RLS 및 JWT 세션 증명 체계를 따름.
- **유지보수성 (Maintainability)**: 아키텍처를 `Client(Front) / Admin / Backend` MSA에 준하는 모노레포 폴더 구조로 철저하게 분리 설계할 것.
