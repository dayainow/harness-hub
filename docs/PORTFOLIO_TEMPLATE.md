## 2. 기획 배경 및 나의 역할

### 💼 프로젝트 기획 개요

> 파편화되어 쏟아지는 수백 개의 AI 도구 중에서 내게 꼭 필요한 도구만을 쉽게 찾고, 그 도구를 직접 사용해 본 사람들의 "활용 방법/실험 레시피(Market)"를 활발하게 공유하는 **'선순환형 AI 지식 생태계 커뮤니티'**를 구축하고자 시작된 플랫폼입니다.

### 📌 핵심 목표 / 요구사항 요약

- 수많은 AI 도구들을 [카테고리], [요금제], [특정 태그] 단위로 교차 검색 및 필터링할 수 있는 UI 환경 지원
- AI 도구를 활용한 결과물(실험실)을 갤러리 피드 형태로 공유하고 실시간으로 공감(Upvote) 투표하는 기능 구성
- 한국어/영어 사용자를 동시 타겟팅하기 위한 다국어(i18n) 번역 라우팅 필수 도입

### 👨‍💻 담당 역할

- **풀스택 프론트/백엔드 단독 개발** (인프라 클라우드 배포 파이프라인 구축 포함)
- URL Params 기반 다이내믹 다중 배열 필터 연산 및 최적화
- Product Hunt 스타일의 투표 버튼 마이크로 인터랙션 (이벤트 라우팅 방어) 설계
- Next.js 서버 컴포넌트(Server Components) 렌더링 최적화 및 외부 CDN 이미지 보안 트러블슈팅

- **실시간 데이터 제어**: 백엔드 Throttle 가드를 통한 비정상 콜 제한 및 프론트엔드의 비동기 데이터 캐싱 구조.

## 5. 문제 해결 및 기술적 도전

### ✅ 문제 사례 1: 외부 이미지 로딩 시 프론트엔드 보안 차단(Broken Image) 현상

> 외부 AI 사이트들의 공식 로고를 실시간 스크래핑해오기 위해 `logo.clearbit.com` CDN을 도입했으나, Next.js의 내장 브라우저 보안 정책에 의해 메인 화면 전체의 이미지들이 로드 거부와 함께 엑스박스 마크로 뜨는 현상 발생. 
> 
> → **해결 방법**: 공식 문서 딥다이브 후 프로젝트 설정 파일인 `next.config.ts`의 `images.remotePatterns` 영역에 위반 사항을 해제하기 위한 호스트네임 화이트리스트(Whitelist)를 등록함으로써, 외부 리소스 제어 보안 정책을 지키면서도 프레임워크 차원에서 지원하는 Webp 엣지 캐싱 이미지 최적화의 압도적인 렌더링 속도 이점까지 모두 취했습니다.

### ✅ 문제 사례 2: 리스트 클릭 라우팅과 이벤트 중첩 전파(Bubbling)로 인한 UX 저하 문제

> 리스트 뷰 내 아무 곳이나 클릭해도 상세 페이지로 예쁘게 라우팅되도록 `Link` 컴포넌트로 감쌌으나, 사용자가 그 내부 엘리먼트인 **'실시간 투표(Upvote) 뱃지'**를 클릭할 때 무조건 부모 컴포넌트 이벤트까지 중첩 발생하면서, 투표 숫자는 올라가기도 전에 상세 페이지로 자꾸 화면이 넘어가버리는 매우 심각한 UX 파괴가 발생했습니다.
> 
> → **해결 방법**: 부모-자식 트리 간의 렌더링 책임을 구조적으로 분리. 투표 컴포넌트인 `<LikeBookmarkButtons>` 만을 클라이언트 컴포넌트(`'use client'`)로 강제 격리시킨 뒤, OnClick 이벤트 내부에서 `e.preventDefault()`와 `e.stopPropagation()` 트리거를 활성화시켜 상위 스택으로 번지는 버블링 체인을 원천적으로 차단시켰습니다. 결과적으로 사용자가 화면 이동 없이도 즉각 빠른 투표 피드백을 받을 수 있게 되었습니다.

### ✅ 문제 사례 3: URL 기반 다중 필터 구조 구현 시 조회 속도 한계

> `?category=개발,오디오&tags=Mac,노코드`와 같은 복잡한 다중 선택 쿼리가 요구됨. 프론트엔드 스토어에서 모든 도구 목록을 가져온 후 JS `filter` 메소드로 거르는 로직은 향후 데이터 10,000건 초과 시 명백한 클라이언트 과부하 병목으로 작용함을 인지.
> 
> → **해결 방법**: 데이터 처리에 대한 모든 책임을 NestJS(백엔드)로 위임. 컨트롤러 단에서 콤마(,) 기준의 요청 구문을 인터셉트하여 배열화시키고, Prisma DB 엔진 고유 기능인 `{ in: [] }` 명령어 및 `{ hasSome: [] }` 배열 스캔 쿼리를 결합. 결과적으로 O(N) 반복문 없이 DB 수준에서 탐색되어 나오는 10ms 이하의 폭발적인 응답 속도 최적화를 구현해 냈습니다.

## 6. 협업 및 커뮤니케이션 (사이드/개인 회고 버전)

- **주도적인 기획 전개**: 기획 초기 벤치마킹 분석부터 모노레포 아키텍처 설계와 컴포넌트 다이어그램까지 마크다운 위키 기반으로 논리적으로 체계화하며 개발 진행.
- **AI 페어 프로그래밍 체화**: 생성형 AI 에이전트와 완벽히 협업하며 백엔드 ORM부터 프론트 상태 관리까지 개발 속도와 코드 퀄리티를 폭발적으로 상향.
- 요구사항 변경(디자인 리스트 뷰 변경, 다중 필터 요구 등)에 당황하지 않고 컴포넌트 유연성을 확보하며 유지보수가 용이한 리팩토링 처리 경험.

## 7. 개발 결과 및 회고

### 📈 결과

- 1인 개발/풀스택 파이프라인(Next.js -> NestJS -> PostgreSQL 연동부터 실배포까지) 전 구축 통합 및 오너십 입증 성공
- 30개 이상의 실제 글로벌 AI 도구 및 태그 데이터 연동으로 실사용이 가능한 최소기능제품(MVP) 수준을 초과 달성

### 🤔 회고

- 백엔드와 프론트엔드를 완벽하게 분리한 클린 아키텍처 관점을 이해하고, Vercel/Render로 찢어져 배포된 서버 간의 CORS 처리 및 에러 로직 관리의 중요성을 크게 체감했습니다.
- Next.js의 가장 핵심인 "서버 컴포넌트"와 "클라이언트 컴포넌트"의 역할을 엄격하게 분배하며 자바스크립트 번들 사이즈를 줄이는 **"Zero-Bundle-Size" 렌더링 최적화** 방식의 매력을 깨달았습니다.
- 다음 프로젝트에서는 현재의 단순 로그인 방식을 넘어 OAuth 기능 고도화와 **데이터 캐싱 구조(Redis)** 를 통한 조회 속도 개선까지 목표해 보고 싶습니다!
