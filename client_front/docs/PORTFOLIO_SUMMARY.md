### **Ola(올라) (Next.js + NestJS) 올인원 AI 커뮤니티 허브 개발**

2026.04 ~ 진행 중

**AI 도구 탐색 플랫폼 풀스택 구현 + Product Hunt 방식의 인터랙티브 실험실 구축 + 다이내믹 다중 필터 최적화 + AI 하네스(Harness) 기반 개발 오케스트레이션**

🏆 **주요 성과**

- **다중 선택 필터링 구조를 '퍼포먼스 향상의 핵심'으로 시스템 단에서 해결 확정**: 수십 개의 태그와 카테고리를 프론트엔드가 배열로 연산하지 않고, Prisma의 `in` 쿼리와 `hasSome` 구문을 결합하여 DB 레벨에서 고도화된 처리로 위임 구축함. 데이터가 폭발적으로 늘어나도 병목 없이 즉각 10ms 단위로 필터링되게 퍼포먼스 아키텍처를 조기 설계함
- **Product Hunt 스타일 리스트뷰를 '실사용 가능한 마이크로 인터랙션'으로 완성**: 리스트 전체를 묶고 있는 공용 라우팅 구간과, 그 안에 배치된 실시간 투표(Upvote) 뱃지 사이의 클릭 이벤트 스택 충돌(Bubbling) 현상을 완벽 방어(`stopPropagation`) 설계함. 이를 통해 유저가 화면 이동 없이도 즉시 좋아요 집계를 보장받는 실사용 위주의 쾌적한 UX를 최종 구현함
- **Next.js 외부 에셋 렌더링 보안 문제를 '안전하고 최적화된 방식'으로 통제**: 30여 개가 넘는 외부 서비스 로고 이미지망(`clearbit CDN` 등)을 들여올 때 걸리는 브라우저 프로토콜 블로킹을 `next.config.ts` 화이트리스트로 분석 및 보안 통과 처리함. 단순 이미지 노출을 넘어 프레임워크 호스팅 단계 자체에서 Webp 리사이징 엣지 최적화를 보장하는 프로덕션 규격으로 마감함
- **프론트/어드민/백엔드 생태계를 '독립적인 MSA 모노레포'로 구축 확정**: 단독 스택으로 타협하지 않고 Client(Vercel) + API(Render) 로드밸런싱 클라우드 구조를 설계하고, Discord 웹훅 알림 파이프라인까지 관제 모니터링 생태계를 완성하여 서비스 운영/개선 루프가 실제 기업 프로덕션 레벨처럼 견고하게 동작하도록 마감함
- **AI 에이전트 하네스(Harness)를 '팀 단위 오케스트레이션'으로 설계·운영**: `CLAUDE.md` 기반으로 백엔드/프론트엔드/통합/QA 전담 에이전트 4개를 분리 구성하고, 애자일 스프린트 단위로 작업을 쪼개 지시하여 환각(Hallucination) 발생률을 최소화함. AI가 생성한 코드에 대해 직접 이벤트 버블링·번들 사이즈 등의 결함을 리뷰하고 역설계 피드백을 제공하는 PM/코드 리뷰어 역할을 수행함

💼 **주요 업무**

- **AI 도구 탐색 검색 엔진 풀스택 구현**: URL Query Params 기반 콤마 스플릿 배열 파서를 프론트에 구현하고, NestJS 컨트롤러에서 Prisma `in`/`hasSome` 복합 쿼리로 매핑하여 카테고리·요금제·태그 다중 교차 필터링 시스템 완성
- **실험실(Market) 피드 및 마이크로 인터랙션 UI 구현**: Product Hunt 벤치마킹 기반의 갤러리형 리스트뷰 설계, Upvote/Bookmark 독립 컴포넌트 분리(`use client` 격리) 및 이벤트 전파 방어 로직 도입으로 Zero-Bundle-Size 최적화 달성
- **인증·프로필·북마크 등 사용자 코어 플로우 구현**: Supabase Auth 기반 Google OAuth 소셜로그인 연동, JWT 세션 검증 미들웨어 구현, 마이페이지 탭별 데이터 패칭 및 에러 핸들링 체계 구축
- **모노레포 Git Subtree 배포 파이프라인 설계·운영**: `client_front` → Vercel, `back` → Render로 각각 독립 배포되는 Subtree Push 기반 CI/CD 파이프라인 구축, 프론트 수정 시 백엔드 무중단 보장하는 MSA 격리 배포 아키텍처 확립
- **AI 하네스(Harness) 오케스트레이션 및 PRD/기술 문서 체계 수립**: 에이전트 팀 역할 분리(백엔드·프론트·통합·QA) 설계, PRD·DB 스키마·유저 플로우·API 명세·트러블슈팅 리포트 등 기획~운영 전 구간 문서 산출물 완성
- **외부 CDN 이미지 보안 최적화 및 i18n 글로벌 아키텍처 구축**: `next.config.ts` remotePatterns 화이트리스트 전략 도입으로 Clearbit/Dicebear 에셋 안전 로딩 확보, `next-intl` 경로 기반 다국어 라우팅 설정으로 글로벌 서비스 확장 기반 마련
