# 🗺️ 핵심 사용자 여정 지도 (Core User Flow)

Ola 서비스를 방문하는 사용자가 "목표를 수행하고 이탈하지 않도록" 어떻게 화면(UX)을 넘어가는지 과정을 매핑한 시나리오 문서입니다.

---

## 🧭 파트 1. 신규 사용자 온보딩 플로우 (Onboarding)

```mermaid
graph TD
    A[랜딩 페이지 진입] --> B{로그인 여부 확인}
    B -- Not Authenticated --> C[회원가입/로그인 모달]
    C --> D[Supabase Auth (Google OAuth)]
    D --> E[세션 생성 및 쿠키 발급]
    E --> F[메인 대시보드 리다이렉트]
    B -- Authenticated --> F
```

- **핵심 포인트**: AI 플랫폼 특성상 번거로운 정보 입력을 배제하고, 소셜 로그인 1-Click 인증으로 즉시 세션을 발급하여 유입 장벽을 완전히 허물었습니다.

---

## 🧭 파트 2. 코어 플로우: 다중 필터링을 통한 AI 도구 탐색 (Discovery)

```mermaid
graph TD
    A[네비게이션: 'AI 도구 탐색' 클릭] --> B[products/page.tsx 렌더링]
    B --> C[좌측 사이드바 필터 패널 노출]
    C --> D[유저 액션: '설정 필터' 중복 체크]
    D -- 예: Mac 태그 + 프리미엄 요금제 --> E[URL 파라미터 업데이트]
    E --> F[서버 컴포넌트 데이터 요청]
    F --> G[알약(Pill) 뱃지 렌더링 및 도구 목록 반환]
    G --> H{결과 리스트 관찰}
    H -- "원하는 도구 발견!" --> I[상세 정보 뷰어 페이지 진입] 
```

- **UX 트러블슈팅**: 다중 속성을 중복 선택할 때 "새로고침" 되는 깜빡임 경험을 없애고 즉각적으로 백엔드 데이터가 치환되는 SSR 구조 동선을 구축했습니다. 필터를 끄는 것도 뱃지 뷰에서 다이렉트(X 버튼) 통신 지원.

---

## 🧭 파트 3. 코어 플로우: Product Hunt형 실험실 투표 플로우 (Engagement)

```mermaid
graph TD
    A[네비게이션: 'AI 실험실' 진입] --> B[피드형 갤러리 리스트 렌더링]
    B --> C{마이크로 인터랙션 (유저 액션)}
    
    C -- "오! 이 레시피 좋네" (👍 투표하기 버튼) --> D[업보트 처리 (API)]
    D --> E[즉각적인 숫자 +1 시각화 피드백]
    
    C -- "나중에 다시 봐야지" (⭐ 북마크 버튼) --> F[Supabase 마이페이지 저장 영역으로 클리핑]
    F --> G[Toast/알림 UI 표출]
    
    C -- "상세 과정 읽기" (리스트 박스 영역 클릭) --> H[해당 게시글 상세 페이지로 라우팅]
```

- **UX/개발 매핑 포인트**: 가장 심혈을 기울인 동선입니다. 게시물 뷰로 들어가지 않아도, **리스트 바깥 화면에서 `Event delegation(버블링)` 충돌 없이 오직 투표만 독립적으로 실행되도록** 컴포넌트를 분리 타겟팅했습니다. 이는 Product Hunt에서 유저들이 하루종일 스크롤을 내리며 투표 버튼만 누르게 만드는 중독성을 정확하게 카피캣한 동선 설계입니다.
